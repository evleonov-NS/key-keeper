# key-keeper

Локальный менеджер лицензий и ключей. Offline, без сервера, с шифрованием данных
мастер-паролем. Хранит лицензионные ключи приложений и подписок, контролирует сроки
их действия и показывает, что активно, что истекает скоро и что просрочено.

> Полное техническое задание — в [PROJECT.md](./PROJECT.md).
> План разработки по этапам — в [PLAN.md](./PLAN.md) (формирует Cursor).
> Ревью UI (мобильный + десктоп): после этапов **6**, **8** и перед **Tauri** — чеклисты в PLAN.
> Текущий статус — в [STATUS.md](./STATUS.md).

---

## Возможности

- Хранение ключей с шифрованием (AES-GCM, ключ из мастер-пароля через PBKDF2).
- Полностью offline — никаких сетевых запросов в рантайме.
- Контроль сроков действия с цветовой индикацией и дашбордом.
- Несколько фото на запись (конвертация в WebP), категории, теги, платформы.
- Зашифрованный бэкап `.vault` и стильный Excel-экспорт.
- Поиск по лицензиям (от 3 символов, учёт раскладки RU/EN, подсветка совпадений).
- Светлая тема по умолчанию, тёмная по переключению. Apple-стиль, минимализм.

**Текущая версия:** `0.7.0` (Этап 7 — логика сроков и статусы). Подробности — [PROJECT.md §10](./PROJECT.md#10-версионирование).

## Технологический стек

- **React + Vite + TypeScript** — UI и сборка
- **Tailwind CSS** — стили (светлая тема по умолчанию, `darkMode: 'class'`)
- **zustand** — состояние (in-memory, этап 1)
- **Dexie.js (IndexedDB)** — локальное хранилище (этап 2)
- **Web Crypto API** — шифрование (AES-GCM + PBKDF2, этап 2)
- **SheetJS (xlsx)** — Excel-экспорт (этап 10)
- **dayjs** — даты и статусы
- **lucide-react** — иконки
- **Tauri 2.x** — десктоп-обёртка (этап 11)

## Установка и запуск

Команды PowerShell построчно (без `&&`). Перед созданием файлов проверять `Get-Location`.

[PowerShell Windows]

    cd F:\Projects\Cursor\Work\key-keeper

[PowerShell Windows]

    npm install

[PowerShell Windows]

    npm run dev

После `npm run dev` открыть адрес из вывода Vite (обычно http://localhost:5173).

### Сборка production-версии

[PowerShell Windows]

    npm run build

[PowerShell Windows]

    npm run preview

## Безопасность — кратко

- Ключи нигде не хранятся в открытом виде: в IndexedDB лежит только зашифрованный блоб.
- Восстановление мастер-пароля **невозможно**. Забыл пароль — данные потеряны.
  Регулярно делайте зашифрованный бэкап `.vault`.
- Авто-блокировка по бездействию (по умолчанию 60 мин, настраивается).
- Excel-экспорт НЕ шифруется — при выгрузке используйте опцию маскировки ключей.

## Конвенция коммитов

Формат: `<type>: <описание по-русски>`, где `type` ∈ `feat`, `fix`, `refactor`,
`chore`, `docs`, `style`.

**При завершении этапа** («ОК», «завершаем этап», «всё ок») Cursor предлагает готовый текст
коммита и команды `git commit` / `git push` — см. [PLAN.md § «Коммит и push»](./PLAN.md#порядок-и-правила-работы).

Структура вехи (этап завершён):

    feat: этап 3 — CRUD лицензий, логин и сессия вкладки (v0.3.1)

    Форма и карточки: создание, редактирование, архив, дубликаты ключа,
    копирование ключа с карточки. Поле accountLogin — открытый текст,
    копирование по клику. Сессия вкладки переживает F5 без повторного
    мастер-пароля. Почта привязки необязательна. Обновлены PLAN, PROJECT, STATUS.

Короткие примеры (мелкие правки внутри этапа):

    fix: корректный расчёт дней до окончания
    docs: обновлён STATUS после этапа 4
