## Проблемы / заметки

### Архитектурные решения (зафиксированы в PLAN.md)

- **Единый зашифрованный блоб:** всё хранилище — один blob. IndexedDB хранит только `encryptedBlob` + `salt` + `verificationBlock`. Лицензии живут в памяти (zustand); поиск/фильтры — in-memory. Любое изменение = перешифровка всего блоба.
- **Этап 1 без персистенции:** демо-данные только в zustand, без Dexie-схемы с открытыми полями. Реальная запись — с Этапа 2, без миграции открытых данных.
- **Импорт `.vault`:** пароль запрашивается от **файла** (своя соль + verificationBlock), не от текущей сессии. После расшифровки — merge/replace и перешифровка текущим мастер-паролем.
- **PBKDF2:** HMAC-SHA-256, ≥600 000 итераций (OWASP).
- **Audit log** перенесён в Roadmap v2 (бывший Этап 11). Tauri — новый Этап 11.

### Прочее

- Репозиторий git ещё не инициализирован — `git init` запланирован в Этапе 0.
- В корне проекта пока только документация — код не создан.
- Web-версия: «постоянно залогинен» **только в Tauri** (Этап 11), не в web.
- Runtime полностью offline — никаких сетевых запросов при работе приложения.
